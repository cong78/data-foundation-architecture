# Support

## Project Status

Version `0.1.0` is the first published, incubating architecture-guidance release for controlled pilots. It is not a supported runtime platform and does not certify an adopter's implementation.

## Request Types

| Request | Route | Include |
| --- | --- | --- |
| Guidance defect | Repository issue tracker | Page or artifact, expected meaning, observed problem, impact. |
| Architecture proposal | Repository issue or pull request | Context, decision, alternatives, affected interfaces, migration, evidence. |
| Security-sensitive guidance issue | Private repository security route | Affected control, exposure, reproduction, impact, safe contact. |
| Adoption question | Adopter's foundation architecture owner | Selected profiles, scope, evidence, decision needed, constraints. |
| Runtime incident | Adopter's service support and incident process | Service, product, environment, severity, trace, contract, current impact. |

Do not place credentials, personal data, customer data, confidential architecture details, or exploitable security information in a public issue.

## Response Expectations

Repository maintainers classify requests by impact and release compatibility. No response-time commitment exists until the repository owner publishes named maintainers and service targets in the hosting platform. Runtime response targets belong to adopter service records and contracts, not this repository.

## Evidence Boundary

Maintainers can validate repository schemas, tests, links, examples, and declared architecture evidence. They cannot approve local risk, product go-live, regulatory compliance, or operational readiness without the adopter's authoritative evidence and accountable decision owners.
